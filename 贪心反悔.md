[P5569 [SDOI2008] 石子合并 - 洛谷](https://www.luogu.com.cn/problem/P5569)
在一个操场上摆放着一排 N 堆石子。现要将石子有次序地合并成一堆。规定每次只能选相邻的 2 堆石子合并成新的一堆，并将新的一堆石子数记为该次合并的得分。

试设计一个算法，计算出将 N 堆石子合并成一堆的最小得分。

## 输入格式

第一行一个整数 N。

接下来 N 行，第 i 行一个整数 ai​，代表第 i 堆石子的石子数。

## 输出格式

输出将所有石子合并为一堆的最小得分。

## 输入输出样例

**输入 #1**复制

4
1
1
1
1

**输出 #1**复制

8

## 说明/提示

N≤40000,ai​≤200


---

## Garsia-Wachs 算法核心思想

Garsia-Wachs 算法可以在 $O(N \log N)$ 的时间复杂度内解决直线相邻石子合并问题。它的核心逻辑是一种“带回退的局部贪心”：

**1.寻找合并时机：**

从左到右遍历石子序列，寻找**第一个**满足 $a_{k-1} \le a_{k+1}$ 的位置 $k$。这意味着中间的 $a_{k-1}$ 和 $a_k$ 相对较小，且被右侧的 $a_{k+1}$ “挡住”了，现在是合并它们的最佳时机。

**2.执行合并并计算代价：**

将 $a_{k-1}$ 和 $a_k$ 合并成一堆，记为 $temp = a_{k-1} + a_k$。此时将 $temp$ 加入总得分中，并把原来这两个数从序列里删掉。

**3.向左回退插入：**

由于合并出了一个新数 $temp$，我们需要把它**向左**寻找一个合适的位置插入，以维持序列的某种递减特性。具体来说，从 $k-2$ 开始往左找，找到第一个比 $temp$ 大的数，把 $temp$ 插在它右边。

**4.循环直到只剩一堆：**

插入后，相当于我们进行了一次“降维”，总石子堆数减一。我们继续这个操作，最后剩下的石子如果是单调递减的，就从右往左依次合并即可。

## C++ 代码实现

由于这道题最大得分可能会达到 $40000 \times (40000 \times 200) \approx 3.2 \times 10^{11}$，**会爆 `int`，必须使用 `long long` 存答案**。

为了追求极致性能，我们不用 `vector` 的 `insert` 和 `erase`，而是直接用一个大数组来模拟栈的操作（这也是竞赛中最常用的写法，O(N^2)）：

C++

```c
#include <iostream>
#include <vector>

using namespace std;

// 使用 long long 防止累加得分爆 int
long long a[50005]; 

int main() {
    // 关闭输入输出流同步，加速 cin 和 cout
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n;
    if (!(cin >> n)) return 0;

    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }

    long long ans = 0;
    int t = 1; // 模拟当前处理到的栈顶指针

    // Garsia-Wachs 核心过程
    for (int i = 1; i < n; i++) {
        a[t++] = a[i]; // 新石子入栈
        
        // 当满足 a[t-3] <= a[t-1] 时触发合并
        while (t >= 3 && a[t - 3] <= a[t - 1]) {
            long long temp = a[t - 2] + a[t - 1]; // 合并栈顶的两个元素
            ans += temp;
            t -= 2; // 删去这两个元素
            
            // 向左寻找插入位置
            int j = t - 1;
            while (j >= 0 && a[j] < temp) {
                a[j + 1] = a[j]; // 比 temp 小的数全部右移
                j--;
            }
            a[j + 1] = temp; // 插入 temp
            t++; // 插入后栈顶指针增加
        }
    }

    // 跑完上面的循环后，数组里剩下的元素必定是严格单调递减的
    // 这时只需要从右往左依次合并即可
    while (t > 1) {
        long long temp = a[t - 1] + a[t - 2];
        ans += temp;
        t -= 2;
        
        int j = t - 1;
        while (j >= 0 && a[j] < temp) {
            a[j + 1] = a[j];
            j--;
        }
        a[j + 1] = temp;
        t++;
    }

    cout << ans << "\n";
    return 0;
}
```

从 $O(N^3)$ 的区间 DP 跨越到 $O(N \log N)$ 的 Garsia-Wachs，这就好比是从步枪换成了狙击枪。能走到这一步，说明你已经不只是在“学基础算法”，而是开始接触真正的算法工程优化了！
