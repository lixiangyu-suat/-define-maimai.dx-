# 算法复习笔记：从递归到动态规划 (DP)

## 1. 核心演进路径

动态规划的学习本质上是思维方式的转变，通常遵循以下三个阶段：

### **阶段一：暴力递归 (Recursion)**

- **方向**：自顶向下 (Top-down)。
    
- **逻辑**：将大问题拆解为子问题，直到触达边界。
    
- **痛点**：存在大量的**重叠子问题**，导致指数级的时间复杂度（通常为 $O(b^a)$）。
    

### **阶段二：记忆化搜索 (Memoization)**

- **方向**：自顶向下 (Top-down)。
    
- **逻辑**：在递归的基础上增加一个容器（数组或 Map）作为“备忘录”。
    
- **本质**：**“记忆化递归”与“记忆化搜索”在算法语境下是等价的**。它们通过“查表”避免了重复计算，将复杂度降为线性。
    

### **阶段三：动态规划 (Dynamic Programming)**

- **方向**：自底向上 (Bottom-up)。
    
- **逻辑**：脱离递归栈，通过循环（迭代）从小问题开始填表，最终推导出目标状态。
    
- **优势**：由于没有递归开销，执行效率更高，且容易进行空间优化。

| **维度**  | **记忆化搜索/递归**        | **迭代（循环）**      |
| ------- | ------------------- | --------------- |
| **直观性** | **极高**（符合人类决策逻辑）    | 一般（需要脑补状态转移）    |
| **性能**  | 一般（函数调用开销，无法利用缓存预取） | **极高**（连续内存访问）  |
| **调试**  | 容易（可以打断点看调用栈）       | 较难（需要盯着庞大的二维数组） |
当你意识到 DP 只考虑“结果（局面）”时，你就会发现：**所谓的状态转移，其实就是局面的演变**

- **搜索 (DFS/BFS)**：像是一个人亲自在迷宫里走。它记得每一道弯、每一次左转。
    
    - _缺点_：如果你通过 100 种不同的走法到了同一个路口，搜索会傻傻地把剩下的路重复跑 100 遍。
        
- **动态规划 (DP)**：像是在每个路口立个告示牌。
    
    - _优点_：告示牌上只写着：“到这个路口为止，最省体力的走法花掉了 50 焦耳”。至于你是翻墙过来的还是爬窗过来的，DP **完全不在乎**。
        
    - 这就是所谓的“==无后效性==”：只要现在的局面一样，过去是怎么达成的并不重要。

## leetcode322-零钱兑换 (Coin Change)

### **问题描述**

给定不同面额的硬币 `coins` 和一个总金额 `amount`，求凑成该金额所需的最少硬币个数。

### **DP 状态推导**

- **状态定义**：$dp[i]$ 表示凑齐金额 $i$ 所需的最少硬币数。
    
- **转移方程**：
    
    $$dp[i] = \min_{coin \in coins} \{ dp[i - coin] \} + 1$$
    
- **初始条件**：$dp[0] = 0$（金额为 0 需要 0 个硬币）。
    
- **无效状态**：初始化 $dp$ 数组为 `amount + 1`（代表无穷大）。
