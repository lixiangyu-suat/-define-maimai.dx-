# 前提条件
- 输入的代价函数a[x]满足上凸函数的性质，那么就符合四边形不等式，那么就可以使用分治优化。

# 如何优化
我们不再需要盲目地遍历所有的 $k$。我们可以使用**分治法（Divide and Conquer）**：

1. 先算出总容量处于中间位置 $mid$ 时的最优分配量 $k_{mid}$。
    
2. 那么对于容量小于 $mid$ 的情况，最优分配量肯定在 $[0, k_{mid}]$ 之间。
    
3. 对于容量大于 $mid$ 的情况，最优分配量肯定在 $[k_{mid}, j]$ 之间。
    

通过这种二分空间的策略，原本需要 $O(m^2)$ 的转移过程，被硬生生地砍到了 **$O(m \log m)$**。

# 动态规划“决策单调性—分治优化”模板

这个模板专门用于解决我们刚才讨论的：**当收益函数满足“上凸（边际收益递减）”等性质，推导出决策点 $opt(j)$ 单调不降时，如何将复杂度从 $O(n \cdot m^2)$ 暴力砍到 $O(n \cdot m \log m)$。**

### 核心模板：分治优化 DP

为了配合滚动数组（空间优化），我们会定义两个一维数组 `olddp` 和 `newdp`。

C++

```c
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 1005;
const int MAXM = 1005;

long long a[MAXN][MAXM];   // 收益矩阵 a[i][k] 表示第 i 个公司分配 k 台机器的收益
long long olddp[MAXM];     // 上一层（前 i-1 个公司）的状态
long long newdp[MAXM];     // 当前层（前 i 个公司）的状态
int current_i;             // 全局变量，记录当前正在处理第几个公司

/**
 * 分治优化计算当前层的 newdp
 * @param L    当前需要计算的容量区间左端点 newdp[L...R]
 * @param R    当前需要计算的容量区间右端点
 * @param optL 这些容量对应的最优决策点 k 的最小可能值
 * @param optR 这些容量对应的最优决策点 k 的最大可能值
 */
void solve(int L, int R, int optL, int optR) {
    // 递归边界：区间非法，直接返回
    if (L > R) return;

    // 1. 找到当前要处理的中间容量 mid
    int mid = L + (R - L) / 2; 
    
    int best_k = optL;      // 用于记录 mid 对应的最优决策点 opt(mid)
    long long max_val = -1; // 记录给前 i 个公司分配 mid 台机器的最大收益

    // 2. 暴力寻找 mid 的最优决策点
    // 关键剪枝：k 的枚举范围被严格限制在了 [optL, min(mid, optR)] 之间！
    // 这就是能把复杂度降下来的核心，不需要从 0 枚举到 mid 了！
    for (int k = optL; k <= min(mid, optR); k++) {
        // 状态转移方程：分配给当前公司 k 台 + 剩下的 mid-k 台留给前面的公司
        long long current_val = olddp[mid - k] + a[current_i][k];
        
        // 更新最大值，并记录是哪个决策点带来的最大值
        if (current_val > max_val) {
            max_val = current_val;
            best_k = k;
        }
    }

    // 记录中间容量的最优解
    newdp[mid] = max_val;

    // 3. 魔法开始：利用决策单调性进行分治！
    // 因为 opt(j) 是单调不降的，所以：
    
    // 对于左半边容量 [L, mid-1]，它们的最优决策点一定在 [optL, best_k] 范围内
    solve(L, mid - 1, optL, best_k);
    
    // 对于右半边容量 [mid+1, R]，它们的最优决策点一定在 [best_k, optR] 范围内
    solve(mid + 1, R, best_k, optR);
}

int main() {
    int n, m; 
    cin >> m >> n; // m 是总容量，n 是物品组数(公司数)

    // 读入数据
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= m; j++) {
            cin >> a[i][j];
        }
    }

    // 初始化 olddp
    memset(olddp, 0, sizeof(olddp));

    // 4. 外层循环：逐个处理每个公司
    for (int i = 1; i <= n; i++) {
        current_i = i;
        
        // 调用分治函数：
        // 计算容量 [0, m] 的状态，初始决策点的范围也是 [0, m]
        solve(0, m, 0, m);

        // 滚动数组：把当前层计算好的 newdp 复制给 olddp，准备进入下一层
        for (int j = 0; j <= m; j++) {
            olddp[j] = newdp[j];
        }
    }

    // 输出最终答案
    cout << olddp[m] << "\n";

    return 0;
}
```

### 模板注释与实战避坑指南

#### 1. 为什么是 `min(mid, optR)`？

在 `for (int k = optL; k <= min(mid, optR); k++)` 这一行中：

- `k <= optR`：这是分治传进来的边界，右半边计算出来的决策上限，不能越界。
    
- `k <= mid`：这是**物理常识**！你现在总共只有 `mid` 台机器，你最多只能给当前公司分配 `mid` 台，绝对不可能分出比总数还多的机器。很多初学者写分治会漏掉这个 `min`，导致 `olddp[mid - k]` 出现负数数组越界。
    

#### 2. 时间复杂度的蜕变

如果你画出分治的递归树：

- 第 0 层：1 个节点，查找范围长度是 $m$。
    
- 第 1 层：2 个节点，它们的查找范围拼接起来（因为边界都是 `best_k`）总长度仍然是 $m$。
    
- 第 2 层：4 个节点，查找范围总和依然是 $m$。
    
- ...
    
    树的深度是 $\log m$。每一层所有 `for` 循环执行的总次数加起来是 $O(m)$。所以**每处理一个公司，时间只花 $O(m \log m)$**。处理 $n$ 个公司，总时间严格稳定在 $O(n \cdot m \log m)$。
    

#### 3. 适用条件的灵魂三问

在比赛中，拿到一个类似背包的题目，不要盲目套这个模板。套用前必须在心里问自己：

1. **这是分组背包/泛化物品吗？** （外层 $i$，内层容量 $j$，决策 $k$）。
    
2. **转移方程长这样吗？** $dp[j] = \max/ \min \{ dp[j-k] + cost(i, k) \}$。
    
3. **最重要：它满足决策单调性吗？** 题目给的数据是具有数学规律的（比如二次函数、或者是明确告诉你边际递减的模型），还是完全随机的黑盒矩阵？如果是黑盒矩阵，**绝不能用此模板**，老老实实写 $O(n \cdot m^2)$ 的暴力。如果是特定的数学模型满足上凸/下凸，直接默写此模板秒杀！

我们这就用最严谨且直观的方式，把这座连接“物理直觉”和“代码优化”的桥梁——**四边形不等式（Quadrangle Inequality, QI）**——彻彻底底地推导一遍。

整个逻辑链条分为三步：

**上凸函数（物理条件）** $\rightarrow$ **四边形不等式（数学表现）** $\rightarrow$ **决策单调性（算法红利）**。

### 第一步：明确我们的战场与变量

在计算分配给当前公司机器数量时，我们要最大化的那个“价值函数”可以写成：

$$w(j, k) = olddp[j-k] + a[k]$$

- $j$：你当前拥有的总机器数（总预算）。
    
- $k$：你决定分给当前公司的机器数（决策点）。
    
- $w(j, k)$：在这个预算和决策下，你能获得的总收益。
    

**核心已知条件：上凸性**

上凸函数（边际收益递减）在离散数学中有一个极其优美的性质：**“内聚大于外散”**。

如果有一个上凸函数 $f(x)$，任取四个点 $x < y \le z < w$，只要它们的坐标之和相等（即 $x + w = y + z$），那么必然有：

$$f(y) + f(z) \ge f(x) + f(w)$$

> **通俗理解：** 两个挨得近的“中庸点”带来的总收益，永远大于一个“极度贫穷”和一个“极度富裕”的点带来的总收益。

_注意：如果 $a[k]$ 是上凸的，那么通过它推导出的最优解数组 `olddp` 也必然是上凸的（这在数学上称为上凸函数的 Max-Convolution 依然是上凸的）。_

### 第二步：上凸性推导“四边形不等式”

现在，我们假设有两个不同的预算：$j_1$（穷）和 $j_2$（富），显然 **$j_1 < j_2$**。

同时，我们有两个不同的分配方案：$k_1$（少给）和 $k_2$（多给），显然 **$k_1 < k_2$**。

我们来看看，在不同的预算下采取不同的方案，它们的收益总和会发生什么奇妙的化学反应。

我们计算交叉组合的差值：

$$\Delta = [w(j_1, k_1) + w(j_2, k_2)] - [w(j_1, k_2) + w(j_2, k_1)]$$

把 $w(j, k) = olddp[j-k] + a[k]$ 代入展开：

前一项：$(olddp[j_1-k_1] + a[k_1]) + (olddp[j_2-k_2] + a[k_2])$

后一项：$(olddp[j_1-k_2] + a[k_2]) + (olddp[j_2-k_1] + a[k_1])$

你会发现，**$a[k_1]$ 和 $a[k_2]$ 被完美抵消了！**

$$\Delta = olddp[j_1-k_1] + olddp[j_2-k_2] - (olddp[j_1-k_2] + olddp[j_2-k_1])$$

现在，见证奇迹的时刻到了。我们观察这四个下标：

设四个点：$x = j_1 - k_2$，$y = j_1 - k_1$，$z = j_2 - k_2$，$w = j_2 - k_1$。

1. 因为 $j_1 < j_2$ 且 $k_1 < k_2$，我们很容易比出大小：**$x$ 是最小的，$w$ 是最大的**（即 $x < y$ 且 $x < z$；$y < w$ 且 $z < w$）。
    
2. 我们把“最小 + 最大”的下标加起来：$x + w = j_1 - k_2 + j_2 - k_1$。
    
3. 我们把中间两个下标加起来：$y + z = j_1 - k_1 + j_2 - k_2$。
    
4. **它们完全相等！** ($x+w = y+z$)
    

根据第一步我们说的“上凸函数内聚大于外散”性质，中间两项的函数值之和，必定大于等于极端的两项之和：

$$olddp[y] + olddp[z] \ge olddp[x] + olddp[w]$$

即：$olddp[j_1-k_1] + olddp[j_2-k_2] \ge olddp[j_1-k_2] + olddp[j_2-k_1]$。

这就意味着我们刚刚计算的 $\Delta \ge 0$。

把 $\Delta$ 还原回去，我们就得到了鼎鼎大名的**一维四边形不等式（交叉大于等于包含）**：

$$w(j_1, k_1) + w(j_2, k_2) \ge w(j_1, k_2) + w(j_2, k_1)$$

### 第三步：四边形不等式推导“决策单调性”

我们拿到这个不等式有什么用？我们对它稍微移个项，把它变成“边际收益”的形式：

$$w(j_2, k_2) - w(j_2, k_1) \ge w(j_1, k_2) - w(j_1, k_1)$$

仔细看这个式子，它的物理意义极其震撼：

等式右边：在穷预算 $j_1$ 的情况下，把机器从 $k_1$ 提升到 $k_2$ 带来的额外收益。

等式左边：在富预算 $j_2$ 的情况下，把机器从 $k_1$ 提升到 $k_2$ 带来的额外收益。

**结论：预算越充足，增加机器配额带来的优势越大（或者说劣势越小）。**

现在，我们用反证法（或者直接推理）来证明决策单调性 $opt(j_1) \le opt(j_2)$：

假设在穷预算 $j_1$ 的时候，经过千辛万苦的打擂台，我们发现**多给机器（$k_2$）比少给机器（$k_1$）更划算**。

用数学语言说就是：$w(j_1, k_2) \ge w(j_1, k_1)$，即 $w(j_1, k_2) - w(j_1, k_1) \ge 0$。

既然穷的时候多给都划算（右边 $\ge 0$），

根据我们推导出的不等式，左边必然更加大于等于 0！

即：$w(j_2, k_2) - w(j_2, k_1) \ge 0 \implies w(j_2, k_2) \ge w(j_2, k_1)$。

**这就是终极铁律：**

如果在预算少的时候，选大的配额 $k_2$ 赢了选小的配额 $k_1$；那么当预算变多时，选小的配额 $k_1$ **绝对不可能**反杀回来赢得第一！

因此，随着预算 $j$ 的不断增加，最优决策点 $opt(j)$ 就像一个只能向前走的棘轮，**永远只能往右滑（单调不降）**。

### 闭环：为什么要用分治？

既然我们从数学上严格证明了：对于 $j_1 < j_2$，必然有 $opt(j_1) \le opt(j_2)$。

那么我们在编程时，就可以理直气壮地切蛋糕了。

当我们算出中间预算 $mid$ 的最优决策是 $opt(mid)$ 时：

对于所有 $j > mid$ 的富裕状态，它们的 $k$ 只要敢小于 $opt(mid)$，就绝对不可能成为最优解（因为上面证明了小配额无法反杀）。

所以我们在代码里，直接把搜索起点砍掉，从 `opt(mid)` 开始循环：

`for (int k = best_k; k <= min(mid, optR); k++)`

这就是从抽象的函数图像（上凸），经过严密的代数变形（四边形不等式），最终化作代码里那句看似简单却威力无穷的剪枝条件（分治优化）的完整过程。
