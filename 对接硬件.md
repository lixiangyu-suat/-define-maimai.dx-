**从 SoC/体系结构优化代码：**

1. **避免 Deep Copy：** 必须使用指针交换或 `std::swap` 代替 `vector` 赋值，减少 $O(C)$ 的 D-Cache 无效填充。
    
2. **数据规模控制：** 若范围允许，优先使用 32-bit 类型，提高 Cache 命中率并降低 DDR 带宽占用。
    
3. **访存对齐：** 意识到跨步访存（Stride $v_i$）在 $v_i$ 较大时会引发频繁的 Cache Miss 和 TLB Miss。
    
4. **预取友好性：** 数组模拟队列比 `std::deque` 具有更好的空间局部性，能更好地触发 CPU 的硬件预取器。

当你使用 `endl` 强行刷新缓冲区时，CPU 必须立刻停下手中的活，发起一个系统调用。

`std::endl`：等于 `\n` + **`std::flush`**。

- **上下文切换**：CPU 要从用户态（你的程序）切换到内核态（操作系统）。这涉及到保存寄存器、切换内存映射等繁琐操作。
    
- **流水线停滞（Pipeline Stall）**：现代 CPU 依靠指令流水线高速运转。系统调用像是一个“紧急刹车”，会导致 CPU 正在预取的指令全部失效，流水线必须排空并重新填充。

ios优化配合 `\n` 使用，速度能提升数倍。（10^4次endl已经明显）
