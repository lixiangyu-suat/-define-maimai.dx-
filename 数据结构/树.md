
| **概念**            | **形象理解**    | **核心数字**              |
| ----------------- | ----------- | --------------------- |
| **重心 (Centroid)** | 树的“平衡点”     | 最大子树 $\le N/2$        |
| **直径 (Diameter)** | 树里“最长的那根绳子” | 树上最远两点的距离             |
| **度数 (Degree)**   | 某个点的“触角”数量  | 总和 $= 2 \times (n-1)$ |
### 重心（Centroid）性质

- **定义：** 如果你删掉一个点，剩下的所有连通块（子树）的大小都不超过 $n/2$，这个点就是重心。
    
- **联系[Problem - 2218F - Codeforces](https://codeforces.com/problemset/problem/2218/F)：** 如果你想要构造一个 $x$ 很大的树（很多偶数子树），你通常需要把树弄得很“长”（链状）；如果你想要 $x$ 极小，你就需要把树弄得很“扁”（星形/葵花形）。

- **数量：** 一棵树要么有 **1个** 重心，要么有 **2个** 连通的重心。
    
    - 如果 $N$ 是奇数，通常只有 1 个。
        
    - 如果 $N$ 是偶数，且重心有两个，它们之间一定有一条边直接相连。
        
- **距离之和：** 重心是树中到所有其他节点**距离之和最小**的点。
    
    - 如果你要在树上盖一个超市，大家都想走最少的路，那超市就应该盖在重心上。
        
- **动态变化：** 给树增加一个叶子节点，重心最多只会向那个叶子方向移动一条边。


### 4. 为什么要学重心？（算法用途）

在处理更难的题目时，重心有一个绝招：**点分治（Centroid Decomposition）**。

- **逻辑：** 1. 找到重心，处理所有经过重心的路径。
    
    2. 删掉重心，树裂成几块。
    
    3. 在每一块里递归找新的重心。
    
- **效果：** 这种“分阶段”处理能把 $O(N^2)$ 的暴力搜索降到 $O(N \log N)$。因为每次找重心都能把规模缩小一半。==(????)==
