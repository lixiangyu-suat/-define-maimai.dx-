## lowbit的计算
- ==将 `x` 的二进制所有位全部取反，再加 1，就可以得到 `-x` 的二进制编码．==例如，6![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "6") 的二进制编码是 `110`，全部取反后得到 `001`，加 `1` 得到 `010`．
```
int lowbit(int x) {
  // x 的二进制中，最低位的 1 以及后面所有 0 组成的数．
  // lowbit(0b01011000) == 0b00001000
  //          ~~~~^~~~
  // lowbit(0b01110010) == 0b00000010
  //          ~~~~~~^~
  return x & -x;
}
```

## 为什么这么划分区间
![[Pasted image 20260502151534.png]]

前缀和查询（比如 $sum(13)$）其实就是把一个数字拆成几个特定的 $2^k$ 块之和。

-  $13 = 8 + 4 + 1$ (正整数二进制表示的唯一性)，这三个数字对应的 $lowbit$ 长度区间，刚好严丝合缝地拼成了从 $1$ 到 $13$ 的全覆盖。
- `12 - lowbit(12) + 1 = 9` 在查询过程中，当前下标减去自己的 $lowbit$ 后，再加 $1$ 得到的那个数，正好就是“上一个大块”结束后的第一个空位。**这意味着减去 $lowbit$ 的动作，本质上是在二进制中**“从右往左依次熄灭 1”的过程.
- 每熄灭一个，就处理完了一个大区间`[x-lowbit[x]+1, x]`。
